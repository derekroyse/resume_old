Representative Tracker - Django Rewrite

This project was previously created with C#/.Net, but after the APIs it relied on were deprecated in late 2018, I decided to rewrite it in Django in order to learn the framework. The site is designed to be a user-friendly solution for users who want a simple and fast way to learn about their politicians and to easily find their latest sponsored bills, committee assignments, vote stances, and news stories.


  • Server-Side: Python/Django
  • Client Side: Javascript/jQuery, Bootstrap
  • Database: Postgres
  • Server: Developed on Windows; pushed to AWS Elastic Beanstalk for hosting.
  • Other tools used: MS Visual Studio, Git, GitHub, Postman, AWS Elastic Beanstalk CLI.

Memory Manager

Part of an operating systems design course, this project served as both a way to learn more about the way an operating system manages memory and as a lesson in the beauty and agony of linked lists. The user can choose between first-fit and worst-fit allocation algorithms and can enable or disable compaction and coalescence.

Languages used: C++
Concepts explored: Linked-lists, first-fit/worst-fit algorithms

Android Restaurant App

This app was another project which involved collaboration with the whole class. Our task was to create an ordering, seating, and point-of-sale system for a (fictional) local restaurant. We met with our 'client', solicited requirements, and split into three teams: Android, Swing, and Networking.
