Roguelike

Roguelike is a simple top down RPG style game featuring elements of randomization and character progression. It features a fully realized inventory system, punishing enemy AI, and a save-game function implemented with SQLite.

My earlier assignments had allowed me to work with a partner, but this assignment was my first opportunity to work with a larger team of seven classmates. I served as the team manager and am very happy with the work we accomplished. This represents our first usage of an external libraries (Curses) and our first use of version control utilities (Git and Github).

Languages used: C++ (w/Curses library)
Concepts explored: Version control (Git/Github), Object-Oriented design

Memory Manager

Part of an operating systems design course, this project served as both a way to learn more about the way an operating system manages memory and as a lesson in the beauty and agony of linked lists. The user can choose between first-fit and worst-fit allocation algorithms and can enable or disable compaction and coalescence.

Languages used: C++
Concepts explored: Linked-lists, first-fit/worst-fit algorithms

Android Restaurant App

This app was another project which involved collaboration with the whole class. Our task was to create an ordering, seating, and point-of-sale system for a (fictional) local restaurant. We met with our 'client', solicited requirements, and split into three teams: Android, Swing, and Networking. I was part of the Networking team and personally wrote code for our server and coordinated work between with the other teams.

This assignment was treated as real-life example of the software design process, and as such we were required to produce all the relevant paperwork such as proposals and design documents. Examples of this are linked here in addition to the code.

Languages used: Java (Swing library)
Concepts explored: Android design, networking sockets, JSON, documentation, client communication
